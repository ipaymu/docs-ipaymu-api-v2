API Merchant Verification berfungsi untuk melakukan pengajuan verifikasi akun iPaymu sebagai
**merchant**. Selain data pribadi, endpoint ini meminta data bisnis/usaha.

## Endpoint

**URL**: `{{baseUrl}}/api/v2/merchant-verification`
**Method**: `POST`

| Lingkungan | URL                                                       |
| :--------- | :-------------------------------------------------------- |
| Sandbox    | `https://sandbox.ipaymu.com/api/v2/merchant-verification` |
| Production | `https://my.ipaymu.com/api/v2/merchant-verification`      |

## Request Header

Endpoint ini mengirim file, sehingga gunakan `Content-Type: multipart/form-data`. Header `va`,
`signature`, dan `timestamp` tetap wajib — lihat [header & signature](/close-api).

## Request Body (multipart/form-data)

### Data Pribadi

| Parameter     | Tipe | Wajib | Deskripsi                                       |
| :------------ | :--- | :---- | :---------------------------------------------- |
| `account`     | text | Ya    | Nomor VA iPaymu yang akan diverifikasi.        |
| `birthday`    | text | Ya    | Tanggal lahir (format `YYYY-mm-dd`).           |
| `birthplace`  | text | Ya    | Tempat lahir.                                   |
| `gender`      | text | Ya    | `L` = Laki-laki, `P` = Perempuan.              |
| `national_id` | text | Ya    | Nomor KTP.                                      |
| `province`    | text | Ya    | Provinsi (dari [API Area](/docs/area-api)).    |
| `city`        | text | Ya    | Kota/Kabupaten (dari API Area).               |
| `district`    | text | Ya    | Kecamatan (dari API Area).                      |
| `village`     | text | Ya    | Kelurahan (dari API Area).                      |
| `postal_code` | text | Ya    | Kode POS.                                       |
| `address`     | text | Ya    | Alamat (maks. 200 karakter).                   |

### Data Bisnis

| Parameter              | Tipe | Wajib | Deskripsi                                                |
| :--------------------- | :--- | :---- | :------------------------------------------------------ |
| `business_name`        | text | Ya    | Nama bisnis/usaha (maks. 50 karakter).                 |
| `business_description` | text | Ya    | Deskripsi bisnis/usaha (maks. 150 karakter).           |
| `business_type`        | text | Ya    | `1` = Personal, `2` = Corporate.                       |
| `business_category`    | text | Ya    | Kategori bisnis (dari [Business Category](/close-api/business-category)). |
| `business_license_no`  | text | Tidak | Nomor izin usaha.                                       |
| `business_address`     | text | Ya    | Alamat bisnis/usaha (maks. 200 karakter).             |
| `business_email`       | text | Ya    | Email bisnis/usaha.                                     |
| `business_phone`       | text | Ya    | No. HP bisnis/usaha.                                    |
| `social_media_url`     | text | Ya    | URL sosial media (`https://example.com/username`).     |
| `website_url`          | text | Ya    | URL website (`https://example.com`).                   |
| `website_ip`           | text | Ya    | IP website (contoh: `127.0.0.1`).                      |

### Data Bank & NPWP

| Parameter      | Tipe | Wajib | Deskripsi                                          |
| :------------- | :--- | :---- | :------------------------------------------------- |
| `bank_code`    | text | Ya    | Kode bank (dari [Bank List](/close-api/bank-list)). |
| `bank_number`  | text | Ya    | Nomor rekening bank.                              |
| `bank_account` | text | Ya    | Nama pemilik rekening (harus sama dengan nama akun/nama bisnis). |
| `npwp_no`      | text | Ya    | Nomor NPWP.                                        |

### Dokumen (File)

| Parameter                | Tipe | Wajib | Deskripsi                                                |
| :----------------------- | :--- | :---- | :------------------------------------------------------ |
| `national_id_photo`      | file | Ya    | File KTP (`.png`/`.jpg`/`.jpeg`, maks. 2MB).           |
| `selfie_photo`           | file | Ya    | Foto selfie dengan memegang KTP (maks. 2MB).          |
| `passbook_photo`         | file | Ya    | Foto/scan buku rekening (maks. 2MB).                  |
| `business_license_photo` | file | Tidak | Scan surat izin usaha (`.png`/`.jpg`/`.jpeg`, maks. 2MB). |

## Contoh Response

```json
{
  "Status": 200,
  "Success": true,
  "Message": "Verification success",
  "Data": {
    "VerificationId": 22567,
    "VerificationStatus": {
      "Code": -2,
      "Desc": "PROCESSING"
    }
  }
}
```

## Keterangan Status Verifikasi

| Code | Desc          | Keterangan                                                |
| :--- | :------------ | :-------------------------------------------------------- |
| `-2` | `PROCESSING`  | Akun dalam proses verifikasi.                            |
| `-1` | `UNCOMPLETED` | Verifikasi ditolak karena dokumen tidak lengkap/tidak valid. |
| `1`  | `VERIFIED`    | Akun sudah terverifikasi.                                |
| `0`  | `BASIC`       | Akun belum terverifikasi.                                |

<Callout type="info">
  `province`/`city`/`district`/`village` diambil dari [API Area](/docs/area-api), `bank_code` dari
  [Bank List](/close-api/bank-list), dan `business_category` dari [Business Category](/close-api/business-category).
</Callout>
