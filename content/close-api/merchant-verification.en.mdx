The Merchant Verification API submits an iPaymu account verification request as a **merchant**. In
addition to personal data, this endpoint requires business data.

## Endpoint

**URL**: `{{baseUrl}}/api/v2/merchant-verification`
**Method**: `POST`

| Environment | URL                                                       |
| :---------- | :-------------------------------------------------------- |
| Sandbox     | `https://sandbox.ipaymu.com/api/v2/merchant-verification` |
| Production  | `https://my.ipaymu.com/api/v2/merchant-verification`      |

## Request Headers

This endpoint uploads files, so use `Content-Type: multipart/form-data`. The `va`, `signature`, and
`timestamp` headers are still required — see [headers & signature](/close-api).

## Request Body (multipart/form-data)

### Personal Data

| Parameter     | Type | Required | Description                                     |
| :------------ | :--- | :------- | :--------------------------------------------- |
| `account`     | text | Yes      | iPaymu VA number to verify.                    |
| `birthday`    | text | Yes      | Date of birth (format `YYYY-mm-dd`).          |
| `birthplace`  | text | Yes      | Place of birth.                                |
| `gender`      | text | Yes      | `L` = Male, `P` = Female.                      |
| `national_id` | text | Yes      | National ID (KTP) number.                     |
| `province`    | text | Yes      | Province (from the [Area API](/docs/area-api)). |
| `city`        | text | Yes      | City/Regency (from the Area API).             |
| `district`    | text | Yes      | District (from the Area API).                  |
| `village`     | text | Yes      | Village (from the Area API).                   |
| `postal_code` | text | Yes      | Postal code.                                   |
| `address`     | text | Yes      | Address (max. 200 characters).               |

### Business Data

| Parameter              | Type | Required | Description                                              |
| :--------------------- | :--- | :------- | :------------------------------------------------------ |
| `business_name`        | text | Yes      | Business name (max. 50 characters).                    |
| `business_description` | text | Yes      | Business description (max. 150 characters).            |
| `business_type`        | text | Yes      | `1` = Personal, `2` = Corporate.                       |
| `business_category`    | text | Yes      | Business category (from [Business Category](/close-api/business-category)). |
| `business_license_no`  | text | No       | Business license number.                               |
| `business_address`     | text | Yes      | Business address (max. 200 characters).               |
| `business_email`       | text | Yes      | Business email.                                        |
| `business_phone`       | text | Yes      | Business phone number.                                 |
| `social_media_url`     | text | Yes      | Social media URL (`https://example.com/username`).     |
| `website_url`          | text | Yes      | Website URL (`https://example.com`).                   |
| `website_ip`           | text | Yes      | Website IP (e.g. `127.0.0.1`).                        |

### Bank & Tax Data

| Parameter      | Type | Required | Description                                        |
| :------------- | :--- | :------- | :------------------------------------------------- |
| `bank_code`    | text | Yes      | Bank code (from the [Bank List](/close-api/bank-list)). |
| `bank_number`  | text | Yes      | Bank account number.                             |
| `bank_account` | text | Yes      | Account holder name (must match the account/business name). |
| `npwp_no`      | text | Yes      | Tax ID (NPWP) number.                             |

### Documents (Files)

| Parameter                | Type | Required | Description                                              |
| :----------------------- | :--- | :------- | :------------------------------------------------------ |
| `national_id_photo`      | file | Yes      | KTP file (`.png`/`.jpg`/`.jpeg`, max. 2MB).            |
| `selfie_photo`           | file | Yes      | Selfie holding the KTP (max. 2MB).                    |
| `passbook_photo`         | file | Yes      | Photo/scan of the bank passbook (max. 2MB).          |
| `business_license_photo` | file | No       | Business license scan (`.png`/`.jpg`/`.jpeg`, max. 2MB). |

## Example Response

```json
{
  "Status": 200,
  "Success": true,
  "Message": "Verification success",
  "Data": {
    "VerificationId": 22567,
    "VerificationStatus": {
      "Code": -2,
      "Desc": "PROCESSING"
    }
  }
}
```

## Verification Status

| Code | Desc          | Description                                               |
| :--- | :------------ | :------------------------------------------------------- |
| `-2` | `PROCESSING`  | Account is being verified.                               |
| `-1` | `UNCOMPLETED` | Verification rejected — incomplete/invalid documents.    |
| `1`  | `VERIFIED`    | Account is verified.                                     |
| `0`  | `BASIC`       | Account is not yet verified.                             |

<Callout type="info">
  `province`/`city`/`district`/`village` come from the [Area API](/docs/area-api), `bank_code` from
  the [Bank List](/close-api/bank-list), and `business_category` from [Business Category](/close-api/business-category).
</Callout>
