This API retrieves the profile details of an iPaymu account by its Virtual Account (VA) number.

## Endpoint

**URL**: `{{baseUrl}}/api/v2/profile`
**Method**: `POST`

| Environment | URL                                         |
| :---------- | :------------------------------------------ |
| Sandbox     | `https://sandbox.ipaymu.com/api/v2/profile` |
| Production  | `https://my.ipaymu.com/api/v2/profile`      |

## Request Headers

See the [headers & signature](/close-api) that every Close API request must include.

## Request Body

| Parameter | Type   | Required | Description        |
| :-------- | :----- | :------- | :----------------- |
| `account` | string | Yes      | iPaymu VA number.  |

## Example Request

```json
{
  "account": "1234567890"
}
```

## Response Parameters

| Field                    | Description                                          |
| :----------------------- | :-------------------------------------------------- |
| `Name`                   | Account holder's name.                              |
| `Email`                  | Account email.                                       |
| `Phone`                  | Account phone number.                               |
| `ApiKey`                 | Account API Key.                                     |
| `Va`                     | Virtual Account number.                             |
| `Gender`                 | Gender (`L` = Male, `P` = Female).                  |
| `Nik`                    | National ID number (KTP).                           |
| `IsMerchant`             | `true` if the account is a merchant.               |
| `MerchantBalance`        | Merchant balance.                                   |
| `MemberBalance`          | Member balance.                                     |
| `VerificationStatus`     | Verification status (see [Member Verification](/close-api/member-verification)). |
| `VerificationStatusDesc` | Verification status description (e.g. `VERIFIED`).  |
| `BankName` / `BankAccount` / `BankNumber` | Account's bank details.            |
| `RegisterDate`           | Account registration date.                         |

<Callout type="warn">
  The response may contain sensitive data (e.g. `ApiKey`, `Nik`). Make sure this data is not written
  to logs and is only processed on a secure server side.
</Callout>
